ARCHI·AI Build Guide · v1

Reverse-engineering the Hypar editor for Archi-AI

A feature map of what makes the reference interface work, decomposed into modules you can build — plus a paste-ready prompt for your Django + TypeScript stack.

Original design. Patterns are generic CAD/3D-editor conventions — not a clone of any branded UI.

The Archi-AI bet

Keyword-parse a plain-English brief → generate a first-pass layout → let non-architects sort, drag, resize, and refine it. The reference tool assumes domain fluency; Archi-AI removes that barrier. Everything below serves that loop.

Keyword parser — the generation core

Split the brief on commas / “and” / newlines into phrases. For each phrase: extract a count (digit or number-word, default 1), match the first keyword against a space-type lexicon, emit N spaces with that type’s default area (± variance), and shelf-pack them into the footprint. Auto-grow the footprint depth to fit.

“12 offices, a lobby, 4 meeting rooms, a kitchen” → 12 Workplace · 1 Lobby · 4 Meeting · 1 Amenity

Ship rules-based first (instant, free, deterministic). Swap the parser for an LLM call later behind the same interface — return the same Space[] shape.

Part 3

The integration prompt

Paste this into your dev workflow (Cursor / Claude Code / etc). It targets a Django + TypeScript codebase and references the two reference designs shipped alongside this guide.

Build the front-end for "Archi-AI", an AI-assisted space-planning
tool for non-architects. Stack: Django (DRF) backend, TypeScript on
the front end, Three.js for the 3D viewport. Match the two reference
designs (Dashboard + Editor) exactly for layout, identity, and tone.

PRODUCT
A user describes a building in plain English. We keyword-parse the
brief into a space program, generate a draft layout in 3D, and let
them refine it by sorting, dragging, resizing, and deleting spaces.
Audience has no architecture background — keep language plain and the
loop forgiving.

VISUAL IDENTITY (do not deviate)
- Background #121110, surfaces #1A1815, ink #F2EEE6, muted #9C968B.
- Single accent: amber #E8A33D. Space-type palette: Workplace
  #E8A33D, Meeting #6FB6A6, Lobby #D98C7A, Amenity #9B8FC7, Lab
  #7FA8D9, Support #8A8378.
- Type: Archivo (UI/headings), IBM Plex Mono (numbers, areas, tags).
- Floating glass panels (blur 12px, 1px hairline border, radius
  9–14px). No gradients on UI chrome, no emoji. Bold and precise.

SCREENS
1. Dashboard: left nav (Projects/Templates/Shared/Site plans/Trash),
   a hero "describe your building" prompt that routes to the editor,
   and a responsive grid of project cards. Each card has a plan-style
   thumbnail (dashed plot boundary + colored space blocks), net area,
   levels, options, last-edited, and presence avatars.
2. Editor: full-bleed Three.js viewport; top bar (project name,
   option switcher, 2D/3D toggle, Perspective + walk, units, live
   "Net Area Placed"); bottom-left AI chat panel; right "Space
   Program" rail; bottom-center tool dock; selection gizmo on click.

VIEWPORT (Three.js)
- Ground GridHelper, hemisphere + 2 directional lights, OrbitControls
  with damping, maxPolarAngle ~ PI/2, preserveDrawingBuffer:true.
- Render the footprint as a thin slab + amber edge outline, each
  space as an extruded BoxGeometry (color by type, opacity 0.9) with
  dark EdgesGeometry. Rebuild a single Group from state on change.
- Click = raycast on pointer-up only if pointer didn't drag (>5px).
- Selected space: amber emissive + amber edges; others dim to 0.5.
- 2D view = top-down locked camera (enableRotate=false); 3D = orbit.

GENERATION (the core)
- Parser: split brief on comma / "and" / newline; per phrase read a
  leading count (digit or word: a/one..twelve, default 1), match a
  keyword lexicon to a space type, emit N spaces sized from the
  type's default area * (0.85–1.15) variance. Cap totals (~90).
- Layout: shelf-pack rectangles left→right within footprint width
  with a 5ft gap, wrap rows, auto-grow footprint depth to fit.
- Show a brief generating state (~0.75s), then animate spaces in and
  frame the camera. Always show "AI drafts a starting point. Verify."
- Keep the parser behind one function returning Space[] so it can be
  swapped for a backend/LLM endpoint later with no UI change.

REFINE
- Spaces panel: grouped by type with color dot, count, net area;
  searchable; clicking a row selects/zooms that type. Re-layout
  (reshuffle + repack) and Report buttons.
- Gizmo: floating label (type dropdown + area + lock) + in-canvas
  handles that resize w/d/h with live dimension chips; edits update
  the model and re-pack neighbours. Drag-reorder + delete in panel.
- Report card: total spaces, space types, net area, gross footprint,
  efficiency %. Export schedule as CSV/PDF.

DATA / API (Django + DRF)
- Models: Project, Option(FK Project), Level(FK Option),
  Footprint(FK Level, polygon), SpaceType(name,color,default_area),
  Space(FK Level, FK SpaceType, w,d,h,x,y).
- Endpoints: CRUD for projects/options/levels/spaces;
  POST /api/projects/{id}/generate {brief} -> Space[] (parser runs
  server-side, persisted to the active Level/Option).
- Front end: typed API client (fetch + TS interfaces mirroring the
  serializers); optimistic updates on drag/resize, debounced PATCH.

QUALITY BAR
- 60fps orbit with ~100 spaces; viewport resizes with the window.
- Keyboard: Enter submits chat (Shift+Enter newline), Esc clears
  selection, Delete removes selected space.
- Accessible: focus states, 44px min hit targets, aria labels on
  icon buttons. Responsive down to a tablet width.
- No "AI slop": no purple gradients, no decorative emoji, no stat
  tiles that don't mean anything. Every element earns its place.

REPO HYGIENE
- Commits and PRs must NOT add Claude / AI as a co-author,
  Co-authored-by trailer, or collaborator. Author = me only.
